Get started

    API Endpoint

        http://127.0.0.1:27800/

AnalogAPI - Controller Support is a Fabric mod for Minecraft that provides comprehensive gamepad functionality with configurable button mappings. It exposes controller state and allows real-time button remapping through a small HTTP API running on localhost.

The server binds to 127.0.0.1:27800 by default. You can override the host and port with the system properties analogapi.host / analogapi.port or with the environment variables ANALOGAPI_HOST / ANALOGAPI_PORT.

All endpoints accept and return application/json. CORS is fully open (Access-Control-Allow-Origin: *) so browser-based dashboards can call the API directly.

Health check


# Liveness probe
curl http://127.0.0.1:27800/api/health

A simple liveness probe that always returns 200 OK while the mod is loaded.
GET http://127.0.0.1:27800/api/health 


Result example :

{
  "ok": true,
  "name": "AnalogAPI"
}

Get movement state


# Read the latest computed movement
curl http://127.0.0.1:27800/api/state

Returns the latest movement state computed by the mod for the current client tick.
GET http://127.0.0.1:27800/api/state 


Result example :

{
  "forward":  0.85,
  "sideways": -0.32,
  "jump":     false,
  "sneak":    false,
  "sprint":   true
}

RESPONSE FIELDS

Field Type Description
forward Float Forward (+) / backward (−) impulse, in [-1.0, 1.0].
sideways Float Right (+) / left (−) impulse, in [-1.0, 1.0].
jump Boolean Whether the jump key is pressed this tick.
sneak Boolean Whether the sneak / shift key is pressed this tick.
sprint Boolean Whether the sprint key is pressed this tick.

Get virtual input


# Check whether a virtual input is currently driving the player
curl http://127.0.0.1:27800/api/virtual-input

Reports whether a virtual input is currently overriding the keyboard, and if so what its values are.
GET http://127.0.0.1:27800/api/virtual-input 


Inactive :

{ "active": false }

Active :

{
  "active":   true,
  "forward":  1.0,
  "sideways": 0.0,
  "jump":     false,
  "sneak":    false,
  "sprint":   true
}

Set virtual input


# Walk forward at full speed while sprinting
curl -X POST http://127.0.0.1:27800/api/virtual-input \
     -H "Content-Type: application/json" \
     -d '{"forward":1.0,"sprint":true}'

Set or replace the virtual input. While set, the mixin uses these values instead of the real keyboard / analog input.
POST http://127.0.0.1:27800/api/virtual-input (PUT is also accepted)

BODY PARAMETERS

Field Type Description
forward Float (optional — default 0) Forward / backward impulse, clamped to [-1.0, 1.0].
sideways Float (optional — default 0) Right / left impulse, clamped to [-1.0, 1.0].
jump Boolean (optional — default false) Whether to hold the jump key.
sneak Boolean (optional — default false) Whether to hold sneak.
sprint Boolean (optional — default false) Whether to hold sprint. 

Result example :

{ "active": true }

Clear virtual input


# Hand control back to the keyboard
curl -X DELETE http://127.0.0.1:27800/api/virtual-input

Removes any active virtual input and restores normal keyboard / libanalog behavior.
DELETE http://127.0.0.1:27800/api/virtual-input 


Result example :

{ "active": false }

Controller API

AnalogAPI - Controller Support provides comprehensive gamepad functionality with configurable button mappings. The controller API allows you to check controller status, manage button bindings, and configure controller settings.

Supported controllers include Xbox, PlayStation, and generic USB gamepads. All buttons can be remapped to any Minecraft action with support for both digital and analog inputs.

Controller Status


# Check controller connection and status
curl http://127.0.0.1:27800/api/controller

Get the current controller connection status and input processing state.
GET http://127.0.0.1:27800/api/controller 


Result example :

{
  "connected": true,
  "enabled": true,
  "driving": false
}

Enable/Disable Controller


# Enable controller input
curl -X POST http://127.0.0.1:27800/api/controller \
     -H "Content-Type: application/json" \
     -d '{"enabled":true}'

Enable or disable controller input processing.
POST http://127.0.0.1:27800/api/controller

Field Type Description
enabled Boolean Whether to enable controller input processing. 

Result example :

{ "enabled": true }

Button Bindings


# Get all button bindings
curl http://127.0.0.1:27800/api/controller/bindings

Retrieve all current button mappings.
GET http://127.0.0.1:27800/api/controller/bindings 


Result example :

{
  "bindings": [
    {
      "button": "a_button",
      "action": "jump",
      "threshold": 0.5,
      "analog": false
    },
    {
      "button": "left_trigger",
      "action": "use",
      "threshold": 0.1,
      "analog": true
    }
  ]
}

Set Button Binding


# Bind A button to jump
curl -X POST http://127.0.0.1:27800/api/controller/bindings \
     -H "Content-Type: application/json" \
     -d '{"button":"a_button","action":"jump"}'

Create or update a button binding.
POST http://127.0.0.1:27800/api/controller/bindings

Field Type Description
button String Controller button (a_button, b_button, x_button, y_button, left_bumper, right_bumper, left_trigger, right_trigger, etc.)
action String Minecraft action (forward, backward, left, right, jump, sneak, sprint, attack, use, inventory, chat, drop_item, pause)
threshold Float (optional) Analog threshold value 0.0-1.0, default 0.5
analog Boolean (optional) Whether this is an analog binding, default false 

Result example :

{ "success": true }

Remove Button Binding


# Remove A button binding
curl -X DELETE http://127.0.0.1:27800/api/controller/bindings \
     -H "Content-Type: application/json" \
     -d '{"button":"a_button"}'

Remove a button binding.
DELETE http://127.0.0.1:27800/api/controller/bindings 


Result example :

{ "success": true }

Controller Configuration


# Get controller configuration
curl http://127.0.0.1:27800/api/controller/config

Get current controller settings including deadzones, sensitivity, and all bindings.
GET http://127.0.0.1:27800/api/controller/config 


Result example :

{
  "leftStickDeadzone": 0.15,
  "rightStickDeadzone": 0.15,
  "triggerThreshold": 0.1,
  "invertYAxis": false,
  "stickSensitivity": 1.0,
  "bindings": [...]
}

Update Controller Configuration


# Configure deadzones and sensitivity
curl -X POST http://127.0.0.1:27800/api/controller/config \
     -H "Content-Type: application/json" \
     -d '{"leftStickDeadzone":0.2,"rightStickDeadzone":0.2,"stickSensitivity":1.2}'

Update controller configuration settings. All fields are optional.
POST http://127.0.0.1:27800/api/controller/config

Field Type Description
leftStickDeadzone Float Left analog stick deadzone 0.0-1.0, default 0.15
rightStickDeadzone Float Right analog stick deadzone 0.0-1.0, default 0.15
triggerThreshold Float Trigger activation threshold 0.0-1.0, default 0.1
invertYAxis Boolean Whether to invert the Y-axis, default false
stickSensitivity Float Stick sensitivity multiplier 0.1-3.0, default 1.0 

Result example :

{ "success": true }

Errors

AnalogAPI returns standard HTTP status codes. The body is always JSON.

Status Meaning
200 Request handled successfully.
400 Invalid JSON body on POST /api/virtual-input. The response includes {"error":"invalid_json","message":"…"}.
404 Unknown path. The response is {"error":"not_found"}.
405 HTTP method not supported on this endpoint. The response is {"error":"method_not_allowed"}.
